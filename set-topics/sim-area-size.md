*No limit on sim.area size* ➡️ resolution is the bottleneck (max 4K, but 2K is ideal)
So the area size and the resolution determines how much data ("how many pixels") we have per square-unit. Say, we have an 1000 x 1000 meters sim area. Our sim resolution is also set to 1000 x 1000 pixel. This means, A sim pixel size in UE world is 1 meter. This might be good, if we view the sim from an airplane. This ain't no good if we walk the sim with a TPS character. So, it is the (sim area size / sim resolution) RATIO, that determines if our output is useful (or not) in a given case.
.
